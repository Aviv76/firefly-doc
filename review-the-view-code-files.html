Review the View Code Files

In this article we'll review:

  • The ShowOrders class is called the Controller
  • the ShowOrdersView class is called the View and is in the "Views" namespace
  • We have the "design view", where we can design our screen using drag and drop
  • We can right click in the "design view" and select "View Code" to see the "Code Behind" of the View. This is where we write code that is relevant to the View (in contrast to code that is relevant to our business logic, which we'll write in the Controller)
  • The relationship between the View's constructor and the controller.
    • When calling the View's constructor, we are sending the controller to it using the this keyword
protected override void OnLoad()
{
    View = () => new Views.ShowOrdersView(this);
}
  • the _controller field in the View is used to access the Controller from the View

Review the different files used in the controller

  • the ShowOrdersView class is called the View and is in the "Views" namespace. (ShowOrders.view.cs)
